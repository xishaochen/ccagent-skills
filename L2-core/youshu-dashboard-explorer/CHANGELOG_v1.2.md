# youshu-dashboard-explorer v1.2 更新日志

**更新时间**: 2026-02-02
**版本**: v1.2

## 🎯 核心改进

### 1. 显示数据模型层级关系

**问题背景**：
有数BI存在两种数据模型：
- **基础数据模型** (DATA_MODEL) - 项目级通用模型
- **报表级数据模型** (NEW_REPORT_DATA_MODEL) - 基于基础模型的定制版本

之前的版本只显示组件直接使用的模型ID，无法看到完整的模型层级关系。

**改进内容**：
现在在显示数据模型信息时，会同时显示：
1. **报表级数据模型ID** - 组件直接使用的模型
2. **基础数据模型ID** (relatedDataModelId) - 如果存在

**示例输出**：
```
📋 数据模型匹配 (1个)

  📊 线索转化监控-【分日线索转化】
     数据模型: 易盾分日线索转化
     模型ID: 289363
     基础模型ID: 102917
     🔗 https://smartwe.youdata.netease.com/dash/folder/450201343?rid=117336&did=244793
```

### 2. 新增 format_model_info 函数

**功能**：
统一格式化数据模型信息的工具函数。

**使用场景**：
- 搜索结果显示
- 看板详情查看
- 任何需要显示模型信息的地方

**函数签名**：
```python
def format_model_info(model: Dict, data_models: Dict) -> str:
    """
    格式化数据模型信息

    Args:
        model: 数据模型对象
        data_models: 所有数据模型的字典（用于查找关联的基础模型）

    Returns:
        格式化的模型信息字符串
    """
```

**输出示例**：
```
易盾分日线索转化 (ID: 289363)
       类型: DATA_MODEL
       基础模型: 易盾分日线索转化 (ID: 102917)
```

## 💡 使用建议

### 何时关注模型层级关系？

1. **数据血缘追踪**
   - 需要了解数据从哪个基础模型来
   - 追踪数据模型的定制历史

2. **模型复用分析**
   - 查看哪些报表使用了同一个基础模型
   - 评估模型变更的影响范围

3. **性能优化**
   - 基础模型的优化会影响所有关联的报表级模型
   - 识别可以合并的重复模型
