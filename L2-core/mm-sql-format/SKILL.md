# MM SQL Format - Hive SQL格式化与注释工具

为猛犸平台Hive SQL生成详细的中文代码注释。

## 输出规范

### 必须输出
输出**格式化后的完整SQL代码**，仅此而已。不要输出中间过程、工具调用日志或分析说明。

### 输出格式

```sql
-- ******************************************************************** --
-- Author: [作者名或"请填写作者"]
-- CreateTime: YYYY-MM-DD
-- UpdateTime: YYYY-MM-DD
-- Comment: [一句话业务说明]
--
-- 【业务说明】
-- 本SQL用于<业务目的>
-- 目标表: <表名> (<表注释>)
-- 数据粒度: <粒度说明>
--
-- 【数据来源】
-- 1. <表名1> - <表注释1>
-- 2. <表名2> - <表注释2>
--
-- 【输出字段说明】
-- <字段名>: <字段注释>
-- ...
-- ******************************************************************** --

-- ============================================================
-- Hive参数设置（如有）
-- ============================================================
-- <参数说明>
set hive.xxx = value;

-- ============================================================
-- 数据插入/主查询
-- ============================================================
<格式化后的SQL语句>
```

## 注释规范

### 1. 语言要求
- 所有注释使用**简体中文**
- 技术术语保留英文（如JOIN、CTE、窗口函数）

### 2. 注释层级
```
-- ******************************************************************** --  -- 文件头边界
-- ============================================================           -- 一级分隔（配置块、主查询块）
-- --------------------------------------------------------              -- 二级分隔（子查询、JOIN块）
-- 普通注释                                                               -- 三级说明
```

### 3. 字段注释格式
```sql
select
    field1,                         -- 字段注释
    field2,                         -- 字段注释
    field3                          -- 字段注释
```

### 4. 各部分注释要求

| 部分 | 必须包含 |
|------|----------|
| **文件头** | 作者、时间、业务说明、数据来源、输出字段 |
| **参数** | 参数用途说明 |
| **子查询** | 业务含义（如"-- 客户基本信息子查询"） |
| **JOIN** | 关联目的（如"-- 关联订单信息"） |
| **窗口函数** | 分组逻辑、排序规则 |
| **CASE WHEN** | 每个分支的业务含义 |

## ⚠️ 核心原则

### 绝对不可改变原始SQL逻辑

| ❌ 禁止 | ✅ 正确 |
|--------|--------|
| 修改WHERE条件 | 只在条件上方添加注释 |
| 改变JOIN顺序或类型 | 保持JOIN原样，仅添加说明 |
| 修改字段名或表名 | 保持原样，在注释中解释 |
| 调整SELECT字段顺序 | 保持顺序，添加行内注释 |

### 验证清单
1. ✅ 所有SQL关键字未被修改
2. ✅ 所有表名、字段名未被改动
3. ✅ 所有条件表达式完整无缺
4. ✅ 括号、引号成对出现

## 示例

### 输入
```sql
set hive.merge.mapredfiles = true;
insert OVERWRITE table yidun_dw.ads_example PARTITION (pt_d = '${pre_1day}')
select t1.userid, t2.username
from table_a t1
join table_b t2 on t1.id = t2.id;
```

### 输出
```sql
-- ******************************************************************** --
-- Author: [请填写作者]
-- CreateTime: 2026-03-18
-- Comment: 用户消费统计
--
-- 【业务说明】
-- 本SQL用于统计用户消费数据
-- 目标表: ads_example (用户消费统计表)
-- 数据粒度: 用户级别
--
-- 【数据来源】
-- 1. table_a - 用户基础表
-- 2. table_b - 用户信息表
--
-- 【输出字段说明】
-- userid: 用户ID
-- username: 用户名称
-- ******************************************************************** --

-- ============================================================
-- Hive参数设置
-- ============================================================
-- 合并小文件，减少输出文件数量
set hive.merge.mapredfiles = true;

-- ============================================================
-- 数据插入
-- ============================================================
insert OVERWRITE table yidun_dw.ads_example
PARTITION (pt_d = '${pre_1day}')

select
    t1.userid,                       -- 用户ID
    t2.username                      -- 用户名称

from table_a t1
-- 关联用户信息表获取用户名
join table_b t2 on t1.id = t2.id;
```

## 注意事项

1. **必须查询表结构** - 使用mammoth-tbl-explorer查询所有涉及的表DDL，
获取**准确的表注释和字段注释**
2. **数据来源注释规范** - 【数据来源】部分的表中文名按以下规则处理：
   - 优先使用查询到的真实表注释
   - 若查询返回"N/A"或空值，**根据SQL语义环境推断**可能的意思
格式：`<表名> - <真实注释（基于语义推测：xxx）>`
3. **保持简洁** - 行内注释控制在20字以内
4. **分区参数** - 注意识别 `${pre_1day}` 等变量参数

### 语义推断示例
- 表别名是 `-- 销售团队` → 推断为销售团队信息表
- WHERE条件包含 `eid=3` → 推断为易盾业务相关表
- 子查询用于关联客户信息 → 推断为客户基础信息表
- 字段包含 `clouduserid` → 推断为客户/账号相关表

### 表结构查询命令
使用 `/mammoth-tbl-explorer` 技能查询表结构
